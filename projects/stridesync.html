// GOAL

Goal

Low-cost wearable gait analysis measuring cadence, ground contact time, flight time, left/right symmetry, knee flexion, and trunk lean, with a post-run dashboard and 3D simulation.

// BUILD

Build

Two ESP32 modules (one per leg) run the architecture, developed in PlatformIO with GitHub for version control. Six MPU-6050 IMUs (thigh, shank, and foot per leg) sit on an I2C bus through a TCA9548A multiplexer and are fused with a Mahony filter, alongside FSR sensors along each leg to track joint angle and foot contact.

A Python dashboard built on Plotly, Pandas, NumPy, and Matplotlib visualizes the run data and 3D gait simulation after each run. Custom SolidWorks housings hold the hardware on the leg.

// THE CALL THAT MATTERED

The call that mattered

The key call was choosing a Mahony filter for orientation instead of raw gyro integration or a full Kalman filter. Cheap MPU-6050s drift badly: integrating gyro rate compounds error over time, while a full Kalman filter is heavy to tune and would not leave headroom on an ESP32 already juggling multiple IMUs over a multiplexed bus in real time.

Mahony sits between the two. It uses the accelerometer's gravity vector as an absolute reference to correct the gyro, and its integral term continuously estimates and removes gyro bias, which is what actually kills the drift. The result is stable orientation at a fraction of a Kalman filter's compute cost, which is what makes real-time multi-IMU capture feasible on this hardware.

// PROGRESS

Progress

The test circuit initializing the MPU-6050s is verified, and the calibration protocol and Mahony filter are implemented and working. The dashboard is complete: key metric graphs, a 3D gait simulation, running data string outputs, and Garmin FIT file comparison, all verified against a generated test run file. Hardware is moving next from breadboard to soldered perfboard modules.
